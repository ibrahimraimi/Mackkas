# Coding Standards

Follow these standards to maintain a consistent and high-quality codebase.

## Naming Conventions
- **Files:** Always use `kebab-case.extension` (e.g., `product-catalog.js`).
- **Variables:** Use `camelCase` in JavaScript and `snake_case` in Python.
- **Classes:** Use `PascalCase` in both Python and JavaScript.

## JavaScript Best Practices
- Use `const` by default, `let` only when necessary.
- Avoid `innerHTML` for user-generated content; use `textContent` instead.
- Use arrow functions for callbacks.
- Log errors gracefully using `console.error`.

## Python Best Practices
- Follow PEP 8 guidelines.
- Use type hints where possible.
- Use `contextlib` or `with` statements for resource management.
- Keep functions small and focused on a single task.

## HTML/CSS
- Use semantic HTML elements (`<nav>`, `<header>`, `<footer>`, `<main>`).
- Keep CSS modular and avoid deeply nested selectors.
- Use variables for colors and common spacing.
